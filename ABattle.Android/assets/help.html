Help

Abattle is a variant of xbattle (UNIX). It is a concurrent multi-player (LAN/WLAN) battle strategy game.

Main Screen

After starting the game you see the main screen. It contains the following buttons:
  • Options
  • Single Player
  • Multi Player
  • Internet Play
  • Impressum

Game Description

The game board is a matrix of hexes which can be occupied by troops of various colors. The number of troops in a cell is indicated by the size of a colored troop circle within that cell.
The troops are commanded by holding the mouse button pressed, move the mouse to the neighbor cell and release the button (or touch the cell and move the finger in the direction you want to establish the movement vector). The command will be acknowledged by the appearance of a movement vector, and thereafter, in each update cycle, a certain proportion of the troops will be moved from the source cell to the destination cell until the source cell is exhausted. Movement vectors can be set in several directions at once, in which case the movement is divided evenly between the vector directions, and the command remains active until cancelled. The movement vector remains active even if the number of troops in that cell is zero. If the source cell is being captured by the enemy, all outgoing movement vectors will be removed.
The game is concurrent, so that commands are given continuously by all players without waiting for turns.

Single Player

Before starting a game, the basic game configuration must be set. The height and width of the board can be set. Higher settings result in smaller cells and maybe not usable on small screens. Also the number of computer players must be selected (the computer plays very stupid!). More options can be chosen with the -button.
The gane starts after pressing the -button. Leave the screen by pressing the -button.

Multi Player


There are two different modes of multi-player game types: Playing with friends within the local network (WLAN, LAN) or with friends over the internet by google play.
Currently only local gaming is working.

A new game can be started by touching the -button. Similar to the single player game, you have to choose the width, height and number of players. Press the -button to start the game. The game will be waiting for the other players to join the game.

Other players can join the game after the game appears in the list on their screen, which may take a while (max 5 sec). The game is displayed with the ip adress and the current number of players. Press the -button to join the game.
After all players have joined, the game will start after 3 seconds.

Options

In the options screen you can select multiple options for the next game These options will be stored after leaving the screen. To reset the options press the -button. These are the options currently existing:

Your name
Name used in the game.


Cell shading Instead of one color for each height level, a color gradient will be used within different levels.
Cone selection The destination cell for a movement vector will be specified by the direction the touch is moving.
Army movement type Simple = The armies will be evenly distributed among the movement vectors and the cells will be processed always in the same order.
Random = The armies will move randomly through the vectors and the cells are being processed randomly.
Levels Number of terrain levels.
Game Updates per s Number of game updates/ticks per seconds to be processed.
Random towns in % Percentage of cells which contain towns.
Armies to win % Number of armies needed to win the game.
Home growth per tick Number of armies armies produced by a home base with each tick.
Town growth per tick
Number of armies produced by a town with each tick.
Base moving armies Number of armies traveling per tick along a movement vector.
Terrain influence in % The extent in which the terrain impacts the movement.
Terrain creator Mode of world creation:
Random = complete random levels
Peaks = distribute a number of peaks in the map (more natural look)
Smooth Peaks = Same as peaks, but less levels.
Peaks in terrain in % Only with terrain creator =peaks: Number of peaks in the map (based on the total count of cells) )
